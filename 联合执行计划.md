# Cyber-Repair-Manual项目 — 联合执行计划 (Human-AI 结对编程)

基于总方案与专门目的规格文档，我已完全理解我们的目标：**在两天的冲刺期内，开发一个"不调大模型 API、纯本地确定性、零幻觉"的Cyber-Repair-Manual系统（思路三）。** 

根据《执行手册》的“Claude Code 使用策略”，**你（Human）负责规则、数据与业务决策，我（AI）负责核心代码、系统搭建与实验图表**。

以下是我们接下来的冲刺计划，分为 5 个阶段，请确认是否可以按此推进：

## 阶段一：基础设施搭建与数据骨架 (Day 1)
**目标**：把骨架搭好，跑通数据流转，解除互相依赖。

- **👤 你的任务：**
  - **冻结 Schema**：在 `data/` 目录下创建 4 个家电分类，每类先写 3-4 条符合 JSONL 规格的种子条目（供我跑通代码用）。
- **🤖 我的任务：**
  - 编写 `requirements.txt` 和一键环境安装脚本。
  - 编写基于 `pydantic` 的 Schema 校验器和加载器（`schema.py`），加入类别分布统计功能。
  - 创建空的 `device_dict.txt` 和 `symptom_dict.txt` 词典骨架。
  - 下载并预缓存本地的 `bge-small-zh-v1.5` TF-IDF 向量模型。

## 阶段二：打通“最薄端到端”链路 (Day 1 下午)
**目标**：保命里程碑。即使只有最简单的功能，项目也能 Run 起来。

- **👤 你的任务：**
  - 补充并完善 `device_dict.txt` 和 `symptom_dict.txt` 词典内容。
  - 继续扩写语料库，目标达到 ~300 条。
- **🤖 我的任务：**
  - **实现理解模块 (`parser.py`)**：使用 `jieba` 加载自定义词典，提取设备、症状词等特征。
  - **实现简易检索 (`retriever.py`)**：先只实现 `BM25` 的初筛召回，暂时不加句向量。
  - **实现生成模块 (`renderer.py`)**：基于模板渲染 Step，加入简易 TextRank 抽取兜底。
  - **搭建 Streamlit 界面 (`app.py`)**：连接上述模块，实现在页面输入"按了没反应"，能输出第一版结构化建议。

## 阶段三：主链路加厚与智能升级 (Day 2 上午)
**目标**：完整实现“思路三”，让系统真正具备 NLP 应用的“复杂性”。

- **👤 你的任务：**
  - 填充各故障条目中的区分性问题（`ask_questions`）。
  - 根据跑通的效果，调整阈值（如检索置信度判定）。
  - 编写独立的测试集 `test_set.jsonl`（50-100条，标 gold label）。
- **🤖 我的任务：**
  - **检索器升级**：在 BM25 之后接入本地 TF-IDF Vector 进行 Cosine 重排。
  - **实现置信度门控 (`decision.py`)**：判断 Top-1 分数与 Top1/Top2 的 Margin。
  - **实现决策树追问**：当置信度低时，加载 JSON 决策树逻辑并提供下钻提问，对接 Streamlit UI。

## 阶段四：评测与消融实验 (Day 2 下午)
**目标**：为报告 Chapter 4 提供充实、漂亮的数字和图表。

- **🤖 我的任务：**
  - 编写 `evaluate.py`，在独立测试集上跑通 BM25-only / Dense-only / 级联 三套配置的 P@1、P@3、MRR，并输出 CSV 与 matplotlib 柱状图。
  - 实现“思路四”的消融对照（TF-IDF + LR 分类器），输出 Accuracy、Macro-F1 与混淆矩阵图。
- **👤 你的任务：**
  - 基于我产出的图表和 CSV，整理报告 Chapter 4 的核心论述。

## 阶段五：系统封板与兜底容错 (Day 2 晚)
**目标**：确保答辩现场万无一失。

- **🤖 我的任务：**
  - 完善 Streamlit 异常捕获（Try/Except）。
  - 处理所有兜底分支：“未找到精确匹配”、“超出 DIY 能力送修”。
  - 编写 README 运行说明文档。
- **👤 你的任务：**
  - 测试 3-4 个必中 Demo 用例与 1 个决策树追问用例，准备答辩演讲。
