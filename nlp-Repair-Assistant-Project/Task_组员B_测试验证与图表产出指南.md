## 🛠️ 操作步骤详解

### 第一步：构建纯净的测试集（防泄漏）
为了测试客观，测试数据必须是系统在训练集里**从来没见过**的表达方式。
1. 在 `data/` 目录下创建（或打开） `test_set.jsonl`。
2. 对照现有的 `train_corpus.jsonl` 中的故障分类，为每个分类编 **2-3 条全新的口语化用户查询**。
3. **格式要求**：测试集的 JSON 只需要这三个字段：
   ```json
   {"query": "我这咖啡机出水简直是一滴滴往外挤，急死人了", "gold_device": "Coffee Machine", "gold_fault_category": "Slow flow or blocked"}
   ```
   *注意：写这些测试句子时，千万不要看 `train_corpus.jsonl` 原文，凭自己脑洞写，这样测试才真实。目标是凑够 50-100 条。*

### 第二步：编写或运行评测脚本 (评估检索模块)
系统现在用的是双重检索（BM25 词汇召回 + 句向量重排）。我们需要证明“双重检索”比“只用BM25”或“只用句向量”更强。
你需要编写一个 `evaluate.py`（如果没把握可以借助 Claude Code 帮你写），它的工作流程是：
1. 载入你刚写的 `test_set.jsonl`。
2. 循环每一条测试语句，分别让系统用三种模式检索：
   - **Mode 1**: 纯 BM25 (`rank_bm25`)
   - **Mode 2**: 纯 Dense Embedding (`sentence-transformers`)
   - **Mode 3**: 级联模式 (我们的方案：BM25 取 Top-10，再用 Dense 重排)
3. 记录命中情况：看“正确答案 (`gold_fault_category`)”排在结果的第 1 位（**P@1**），还是前 3 位（**P@3**），并计算 **MRR** (Mean Reciprocal Rank)。

### 第三步：生成分析图表（为报告 Chapter 4 输送炮弹）
数字不够直观，老师喜欢看图。在你的 `evaluate.py` 中，使用 `matplotlib` 或 `seaborn` 将结果画出来：
1. **对比柱状图**：
   - 横坐标：三种检索方法（BM25, Dense Only, Cascaded）
   - 纵坐标：准确率百分比。
   - 画出 P@1 和 P@3 的分组柱状图。
2. **（可选加分项）分类器消融对比（Ablation）**：
   - 用 `scikit-learn` 写一个最简单的 TF-IDF + Logistic Regression 的文本分类器，拿 `train_corpus` 训练，在 `test_set` 预测。
   - 把它的准确率也加到上面的图表里，用作我们**“废弃方案”**的对照。
   - 用 sklearn 画一个 **混淆矩阵 (Confusion Matrix)** 的热力图，看看模型最容易把哪两种故障搞混（这个图放到报告里极其加分，是非常好的 Discussion 题材）。

### 第四步：保存与规整
- 所有的评测指标导出为一个 `results.csv`。
- 所有的图表保存为高分辨率的 `retrieval_comparison.png`, `confusion_matrix.png`。
- 将这些产物统一放在一个名为 `eval_results/` 的文件夹里。
