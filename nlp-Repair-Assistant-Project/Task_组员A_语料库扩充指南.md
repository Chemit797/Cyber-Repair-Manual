## 🛠️ 操作步骤详解

### 第一步：找什么语料？从哪里找？
我们目前的设备包含：`Flashlight`（手电筒）, `Kettle`（电水壶）, `Coffee Machine`（咖啡机）, `Fan`（电风扇）。
你可以：
1. **为现有设备增加新的故障类别**（比如咖啡机的“蒸汽棒堵塞”或“按键失灵”）。
2. **（如果有商量好的新设备）增加新设备的故障**，例如 `Microwave`（微波炉）等。

**语料来源推荐**：
- **官方说明书 (Manuals)** 的 Troubleshooting（故障排查）章节。
- **iFixit** 的维修指南和用户提问。
- **Reddit / Quora / 知乎 / 百度知道** 等论坛中用户的真实提问（这非常重要，因为这里有最真实的“大白话”症状描述）。
- 家电维修博客或 YouTube 视频下方的评论区。

### 第二步：按什么格式写？（核心重点）
我们的数据都在 `data/train_corpus.jsonl` 中。**这是一个 JSONL 文件，意味着每一行都是一个独立的 JSON 对象，不能换行。**

请完全按照以下格式进行扩充：

```json
{"id": "device_fault_001", "device": "Device Name", "fault_category": "Short Fault Name", "symptom_text": ["大白话描述1", "描述2", "描述3..."], "severity": "DIY Repair 或 Requires Professional Service", "possible_causes": ["原因1", "原因2"], "ask_questions": [{"q": "为了区分其他故障的询问词？", "yes": "是的结果", "no": "否的结果"}], "solution_steps": ["步骤1", "步骤2"], "tools": ["螺丝刀", "万用表"], "est_time_min": 15, "safety_notes": ["安全警告1"]}
```

#### 关键字段要求（必读）：
1. **`symptom_text`（非常关键）**：
   - 必须写 **10 - 15 句**不同的、**极度口语化**的用户提问。
   - 不要写“设备电源指示灯未亮起”，要写：“插上电毫无反应”、“像死机了一样”、“按了灯不亮”。
   - 这是系统能匹配到用户输入的唯一凭证！越多、越贴近真实口语越好。
2. **`severity`（严重等级）**：
   - 只能填 `DIY Repair`（可自己修）或者 `Requires Professional Service`（强电/拆机高危操作，必须送修）。如果是送修，步骤里必须体现安全警告。
3. **`ask_questions`（决策树追问）**：
   - 如果这个故障和另一个故障很像（比如风扇电机卡死 vs 电容坏了，都会嗡嗡响），在这里写一个能区分它们的询问词。如果不易混淆，可以留空 `[]`。
4. **`solution_steps`（解决步骤）**：
   - 必须是**原子化、按先后顺序**的列表。比如 `["第一步断电", "第二步拆开后盖", "第三步检查电容"]`。

### 第三步：如何测试你写的数据？
写完后，你需要验证自己写的 JSON 格式对不对，会不会导致程序崩溃。
你可以自己写一个简单的 python 脚本，或者直接运行我们的数据加载代码验证：
```python
import json
with open('data/train_corpus.jsonl', 'r', encoding='utf-8') as f:
    for line in f:
        json.loads(line) # 如果没报错，说明格式没问题
```
（确保括号和引号全部是**英文半角**！）
